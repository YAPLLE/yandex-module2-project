# Анализ лояльности пользователей Яндекс Афиши с помощью Python

В этом проекте вы будете работать с реальными данными сервиса **Яндекс Афиша**. Он позволяет пользователям узнавать о мероприятиях в разных городах и покупать на них билеты. Яндекс Афиша сотрудничает с организаторами мероприятий и билетными операторами, которые публикуют информацию и выставляют билеты на продажу.

Команда маркетинга поставила перед собой цель: не просто привлекать новых пользователей, а удерживать их и превращать в постоянных. Для этого нужно повысить долю клиентов, которые после одной покупки приходят снова и совершают новые.

Вам нужно помочь команде маркетинга лучше понять поведение пользователей. Для этого проведите исследовательский анализ данных, чтобы выяснить, какие пользователи с большей вероятностью возвращаются на платформу и делают заказы повторно. Это позволит:

- Быстро выявлять перспективных клиентов и предлагать им персонализированные условия.
- Точно настраивать рекламу на аудитории с высокой вероятностью возврата.
- Оптимизировать маркетинговые бюджеты.
- Повысить общий уровень удержания клиентов.

**Исследовательский анализ данных** — ключевая часть работы специалистов по Data Science. На этом этапе они изучают структуру и содержание данных, выявляют взаимосвязи между переменными и готовят обоснования для принятия решений.

Исследовательский анализ данных может быть как подготовкой к построению моделей, так и самоцелью: например, для проверки бизнес-гипотез или выявления закономерностей. EDA помогает специалистам по Data Science сформировать обоснованное понимание предметной области и выбрать корректные направления для дальнейшей работы.

## Структура проекта

Проект состоит из **пяти частей**, которые последовательно проведут вас от подключения к данным до финальной публикации работы.

1. **Знакомство с данными.** Вы самостоятельно подключитесь к базе данных SQL и проведёте первичное знакомство с данными, ответив на несколько вопросов.
2. **Подготовка к выгрузке данных.** Вы решите несколько SQL-задач в тренажёре. Они помогут вам подготовить запрос для выгрузки данных, которые вы потом будете анализировать инструментами Python.
3. **Исследовательский анализ данных.** На этом этапе вы пройдёте несколько шагов:
    - Выгрузите данные из БД SQL в датафрейм pandas, чтобы выполнять дальнейшую работу с помощью Python и библиотек анализа данных.
    - Выполните предобработку данных.
    - Сформируете профиль пользователя, выделив признаки и факторы, которые могут влиять на повторные заказы.
    - Выясните, как распределены данные и какие закономерности в них наблюдаются.
    - Проведёте корреляционный анализ и вычислите коэффициенты `phik`, чтобы узнать, какие признаки влияют на количество заказов.
4. **Выводы.** Опираясь на результаты исследовательского анализа данных, вы сформулируете рекомендации для бизнеса.
5. **Публикация проекта в Git.** Подготовите проект для публикации его в репозитории Git.
